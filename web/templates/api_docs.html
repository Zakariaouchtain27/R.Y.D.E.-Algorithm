RYDE Developer API

Submit your clients’ bookings. PRISM monitors prices 24/7 and automatically rebookings when savings exceed your cancellation fee.

REST JSON HMAC Webhooks 60 req/min

Quick Start

Base URL for all requests:

{{ base_url }}/api/v1

Every request needs your agency API key in the header:

X-API-Key: ryde_live_yourslug_a3f9c2d1...

Submit a booking in one call:

curl -X POST {{ base_url }}/api/v1/monitor \
  -H "X-API-Key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "origin": "JFK",
    "destination": "CDG",
    "departure_date": "2026-09-15",
    "original_price": 850.00,
    "cancellation_fee": 75.00,
    "fare_type": "refundable",
    "cabin_class": "economy",
    "webhook_url": "https://yourapp.com/webhooks/ryde"
  }'
RYDE charges a 20% success fee on realized savings only. No saving = no charge.

Authentication

Pass your API key in the X-API-Key header on every request. Keys look like:

ryde_live_acmetravel_a3f9c2d1b4e8f7c2d9a1b3e5f8c4d2a7

Contact api@ryde.io to get your key. Keep it secret — it grants full access to your agency’s bookings.

Errors & Rate Limits

StatusMeaning
200 / 201Success
401Missing X-API-Key header
403Invalid or revoked API key
404Booking not found (or belongs to another agency)
422Validation error — check the detail field
429Rate limit exceeded — retry after 60 seconds
500Server error — contact support

Rate limit: 60 requests per 60 seconds per API key. The 429 response includes a Retry-After: 60 header.

Non-refundable fares are rejected. Only submit refundable or partially_refundable fares. Submitting a fare where cancellation_fee ≥ original_price also returns 422.

Endpoints

POST /api/v1/monitor
Submit a booking for PRISM monitoring. Returns a tracking_id you use for all subsequent calls.

Request body

FieldTypeRequiredDescription
originstringyesIATA code, e.g. JFK
destinationstringyesIATA code, e.g. CDG
departure_datestringyesFuture date in YYYY-MM-DD
original_pricenumberyesPrice paid in USD (must be > 0)
cancellation_feenumberyesAirline fee to cancel and rebook (≥ 0, < original_price)
fare_typestringyesrefundable or partially_refundable
cabin_classstringnoeconomy (default) | premium_economy | business | first
webhook_urlstringnoHTTPS URL to receive decision events
referencestringnoYour internal booking reference

Idempotency

Pass Idempotency-Key: <uuid> to safely retry on network errors. The same key always returns the same tracking_id.

GET/api/v1/bookings
List all bookings for your agency. Optional query param: ?status_filter=monitoring or stopped.
curl {{ base_url }}/api/v1/bookings \
  -H "X-API-Key: YOUR_KEY"

Webhook Events

When you provide a webhook_url, RYDE POSTs a signed JSON payload whenever PRISM makes a decision or executes a rebooking.

ryde.decision

{
  "event": "ryde.decision",
  "booking_id": "b2b_3a9f2c1d4e8b7a05",
  "action": "WAIT",
  "confidence_score": 67.4,
  "net_savings": 180.00,
  "probability_of_future_drop": 0.73,
  "seat_urgency_multiplier": 1.1,
  "reasoning": "Price likely to drop further. Holding."
}

Possible action values:

ActionMeaning
WAITMonitoring, no action taken
STRIKERebooking is being executed now
PHANTOM_HOLDPrice locked, evaluating
IGNORESaving not worth the fee

ryde.rebooking

{
  "event": "ryde.rebooking",
  "booking_id": "b2b_3a9f2c1d4e8b7a05",
  "success": true,
  "old_ref": "ABC123",
  "new_ref": "XYZ789",
  "savings_realized": 142.50,
  "error": null,
  "timestamp": "2026-05-11T14:01:00"
}

Webhook Signature Verification

Every webhook includes an X-RYDE-Signature header. Always verify it to reject spoofed requests.

Python

import hashlib, hmac

def verify_ryde_webhook(body: bytes, header: str, secret: str) -> bool:
    expected = hmac.new(secret.encode(), body, hashlib.sha256).hexdigest()
    return hmac.compare_digest(header, f"sha256={expected}")

# FastAPI / Starlette example
@app.post("/webhooks/ryde")
async def handle(request: Request):
    body = await request.body()
    sig  = request.headers.get("X-RYDE-Signature", "")
    if not verify_ryde_webhook(body, sig, RYDE_WEBHOOK_SECRET):
        raise HTTPException(status_code=401, detail="Invalid signature")
    payload = json.loads(body)
    ...

Node.js

const crypto = require('crypto');

function verifyRydeWebhook(body, header, secret) {
  const expected = 'sha256=' + crypto
    .createHmac('sha256', secret)
    .update(body)
    .digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(header), Buffer.from(expected)
  );
}
Use hmac.compare_digest / timingSafeEqual — not ==. Constant-time comparison prevents timing attacks.
